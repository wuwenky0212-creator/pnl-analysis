# ✅ 公网部署检查清单

## 📋 部署前检查

- [ ] GitHub账号已注册
- [ ] 代码已推送到GitHub仓库
- [ ] Railway账号已注册（或选择其他平台）

## 🚂 Railway部署步骤

### 后端部署

- [ ] 在Railway创建新项目
- [ ] 连接GitHub仓库
- [ ] 等待Docker构建完成
- [ ] 配置环境变量：
  - [ ] PORT=8002
  - [ ] HOST=0.0.0.0
  - [ ] DEBUG=False
  - [ ] DATA_FORMAT=mock
- [ ] 生成后端域名
- [ ] 复制后端URL（例如：https://xxx.railway.app）

### 前端部署

- [ ] 在Railway创建新服务（或使用Vercel）
- [ ] 配置Root Directory为 `frontend`
- [ ] 在 `frontend/index.html` 中添加后端URL meta标签
- [ ] 提交代码到GitHub
- [ ] 等待重新部署完成
- [ ] 获取前端URL

### 测试

- [ ] 访问前端URL
- [ ] 检查页面是否正常加载
- [ ] 打开浏览器控制台（F12）
- [ ] 检查API请求是否成功
- [ ] 测试功能是否正常

## 🎉 完成！

- [ ] 前端可以访问
- [ ] 后端API可以访问
- [ ] API文档可以访问（/docs）
- [ ] 功能测试通过
